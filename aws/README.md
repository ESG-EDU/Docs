## (1) EC2란

- **EC2 (Elastic Compute Cloud)**는 AWS에서 제공하는 **클라우드 기반 가상 서버** 서비스입니다.
- 사용자가 필요할 때 **서버를 생성, 실행, 중지, 삭제**할 수 있으며, 물리적 서버를 직접 관리할 필요가 없습니다.
- 서버를 **필요한 만큼 확장하거나 축소**할 수 있는 특징 때문에 ‘Elastic’이라는 이름이 붙었습니다.
- 대표적인 사용 사례:

  - 웹 서버, 애플리케이션 서버 운영
  - 데이터 처리 및 분석
  - 배치 작업, 머신러닝 학습
  - 테스트/개발 환경

## (2) EC2 구성 요소

> 2-1. 인스턴스(Instance)

- EC2에서 **가상 서버 자체**를 말합니다.
- 생성 시 **운영체제(OS), CPU, 메모리, 스토리지** 등을 선택할 수 있습니다.
- 인스턴스 유형(Type)에 따라 성능이 달라집니다.

  - 예: t2.micro, m5.large, c6g.xlarge 등

> 2-2. AMI(Amazon Machine Image)

- **EC2 인스턴스를 만들기 위한 OS 템플릿**
- 예: Ubuntu, Amazon Linux, Windows Server 등
- 필요한 소프트웨어가 포함된 **커스텀 AMI**도 생성 가능

> 2-3. 인스턴스 유형(Instance Type)

- CPU, 메모리, 네트워크 성능 기준으로 나뉩니다.
- 주요 유형:

  - **General Purpose (t3, t4g)**: 균형 잡힌 성능, 웹 서버/소규모 DB
  - **Compute Optimized (c5, c6g)**: 계산 작업 최적화, 게임 서버, HPC
  - **Memory Optimized (r5, r6g)**: 메모리 집중, 인메모리 DB, 캐싱
  - **Storage Optimized (i3, d2)**: 디스크 성능 중요, 빅데이터 분석

> 2-4. EBS(Elastic Block Store)

- EC2의 **영구 블록 스토리지**
- 인스턴스 종료 후에도 데이터 유지 가능
- 유형:

  - **gp3 / gp2**: 일반 용도 SSD
  - **io2**: 고성능 SSD
  - **st1 / sc1**: HDD 기반, 저비용 대용량

> 2-5. Security Group

- EC2의 **방화벽 역할**
- 인바운드/아웃바운드 트래픽을 제어
- 예: HTTP(80), HTTPS(443), SSH(22) 포트 열기

> 2-6. Key Pair

- **SSH 접속용 인증 키**
- Private Key를 통해만 인스턴스에 접근 가능

## (3) EC2 인스턴스 생성 과정

1. **AMI 선택** → OS 선택
2. **인스턴스 유형 선택** → 성능과 비용 결정
3. **스토리지 설정** → EBS 용량/유형 지정
4. **보안 그룹 설정** → 접속 포트 정의
5. **Key Pair 선택 또는 생성** → SSH 접속 인증
6. **인스턴스 시작** → 공인 IP로 접속 가능

## (4) EC2 관리

- **실행/중지/종료**

  - 실행: 서버 시작
  - 중지: 비용 절감 가능 (스토리지 비용만 발생)
  - 종료: 서버 삭제, 데이터 사라짐

- **Auto Scaling**

  - 트래픽에 따라 자동으로 인스턴스 수를 늘리거나 줄이는 기능

- **Elastic IP**

  - 고정 공인 IP 제공, 인스턴스 재시작에도 변하지 않음

## (5) 비용 구조

- **온디맨드(On-Demand)**: 시간 단위 사용 요금
- **예약 인스턴스(Reserved)**: 1~3년 약정 시 할인
- **스팟 인스턴스(Spot)**: 남는 용량 저렴하게 사용, 중단 가능
- 주요 요인: 인스턴스 유형, 시간, 스토리지, 데이터 전송

## (6) 장점과 단점

> 장점

- 필요할 때 즉시 서버 생성 가능
- 초기 비용 없이 사용 가능
- 유연한 확장성
- 다양한 OS와 인스턴스 유형 지원

> 단점

- 장기적으로 사용하면 비용이 높을 수 있음
- 관리자가 직접 보안/백업 신경 써야 함
- 복잡한 네트워크 구성 시 학습 필요
