## (1) **Prompt Engineering이란**

> **Prompt Engineering**은 **AI 언어 모델(예: ChatGPT, Claude, Gemini 등)**이 원하는 결과를 출력하도록 **프롬프트(입력 문장)를 설계·조정하는 기술**입니다.
- 즉, “AI에게 지시하는 방식”을 최적화하는 기술입니다.

> AI 모델은 다음 두 가지에 크게 의존합니다:

1. **입력(prompt)의 품질**
2. **모델의 구조 및 학습 데이터**

> 같은 모델을 사용하더라도 **프롬프트 작성 능력에 따라 결과물이 완전히 달라질 수 있습니다**.

## (2) 왜 Prompt Engineering이 중요한가?

> AI 모델은 ‘지시’를 기반으로 동작

- 모델은 사용자의 프롬프트에 따라 해석·추론·생성 깊이를 조정합니다.

> 정확한 지시 → 정확한 결과

- 불명확한 프롬프트는 엉뚱한 답변, 부정확한 정보, 불필요한 내용 등을 유발합니다.

> 실무 생산성 극대화

- 코드 생성 / 디버깅
- 데이터 분석
- 마케팅·카피라이팅
- 교육 / 학습
- 자동화된 업무 프로세스
- 콘텐츠 제작

## (3) Prompt Engineering의 핵심 원리

> **① 명확성 (Clarity)**

- AI가 ‘모호한 표현’을 가장 싫어합니다.
- 맥락, 목적, 범위를 명확히 제시해야 합니다.

> **② 구조화 (Structure)**

- 프롬프트 내부를 **역할(ROLE) → 지시(TASK) → 컨텍스트 → 예시 → 형식 지정** 순으로 구조화하면 성능이 좋아집니다.

> **③ 구체성 (Specificity)**

- 정보가 구체적일수록 더 정확한 결과가 나옵니다.
```
예: “좋은 자기소개서 써줘”
→ “경영학과 지원자, 팀 프로젝트 경험 강조, 400~500자, 한국 기업 스타일”
```

> **④ 예시 제공 (Few-shot prompting)**

- 예시를 넣으면 모델은 패턴을 학습하여 출력 품질이 상승합니다.

> **⑤ 출력 형식 정의 (Output formatting)**

- 원하는 출력 형식(JSON, bullet, 표)을 명확히 지정하면 일관성이 뛰어납니다.

> **⑥ 반복과 개선(Iterative prompting)**

- 한 번에 완벽한 결과를 기대하기보다 수정 요구 → 개선 반복 구조가 효과적입니다.

## (4) 대표적인 Prompt Engineering 기법

> **① Role Prompting (역할 부여)**

```
너는 지금부터 경험 많은 마케팅 전문가야.
```
- 역할을 명시하면 모델의 관점·문체·분석 방향이 크게 바뀝니다.

> **② Chain-of-Thought (연쇄 사고)**

- 모델에게 **생각 과정을 공개적으로 적도록 유도**하는 기법
- 예:
```
문제 해결 과정을 단계별로 논리적으로 설명하고 마지막에 결론을 제시해줘.
```

> **③ Few-shot Prompting (사례 제공)**

- 예시를 1~5개 넣으면 모델의 결과 품질이 향상됨.

> **④ Zero-shot Prompting (예시 없는 고도 지시)**

-예시 없이 명확한 지시만 사용.
- 예:
```
다음 글을 한국 고등학생이 이해할 수준으로 쉽게 바꿔줘.
```

> **⑤ Self-Consistency Prompting**

- 여러 답변을 만들게 하고 가장 설득력 있는 답을 선택.

> **⑥ Rewriting Prompt (프롬프트 재작성)**

- AI에게 **더 좋은 프롬프트를 작성하게 하는 기법**도 매우 강력함.
- 예:
```
이 작업을 가장 잘 수행할 수 있도록 프롬프트를 전문가 수준으로 다시 작성해줘.
```

> **⑦ Output specializing (출력 특화)**

- 형식, 톤, 길이, 난이도 등을 강제 지정.
- 예:

```
형식:
1. 핵심 요약(3줄)
2. 상세 설명(300자)
3. 예시(2개)
```

## (5) Prompt의 기본 템플릿

> **프롬프트 템플릿**

```
[역할(Role)]
당신은 최고의 ○○ 전문가입니다.

[목적(Objective)]
이 프롬프트의 목적은 ○○하는 것입니다.

[배경(Context)]
- 대상 독자:
- 사용 목적:
- 요구 사항:

[작업(Task)]
아래 내용을 수행하세요:
1.
2.
3.

[출력 형식(Output Format)]
- bullet 형식 / 표 / JSON 등

[제한 조건(Constraints)]
- 글자 수
- 톤 & 스타일
```

## (6) 예시 비교: 나쁜 프롬프트 vs 좋은 프롬프트

> 나쁜 프롬프트
```
마케팅 아이디어 추천해줘.
```

> 좋은 프롬프트

```
당신은 SNS 바이럴 마케팅 전문가입니다.
10~20대 여성 대상의 뷰티 브랜드가 인스타그램에서 실행할 캠페인 아이디어 5가지를 제안하세요.
각 아이디어는
- 핵심 컨셉
- 예상 해시태그
- 기대 효과
를 포함하세요.
```
- 훨씬 구체적이고 실행 가능.

## (7) 실무에서 많이 쓰는 고급 Prompt Engineering 패턴

- **TAP 방식 (Task–Action–Persona)**
- **RICCE Framework (Role–Input–Context–Constraints–Examples)**
- **APE 방법 (Ask–Provide example–Enforce format)**
- **CRISP 방법(Capability–Role–Instruction–Steps–Parameters)**

## (8) Prompt Engineering을 잘하는 사람의 특징

- 작업 목적을 명확하게 정의한다
- AI가 잘하는 것/못하는 것을 이해한다
- 반복하여 프롬프트를 개선한다
- 출력 오류를 분석하고 “어떤 지시가 부족했는지” 역추적한다
- 템플릿을 만들어 재사용한다
